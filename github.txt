git基本命令：
	git config --list 查看当前git的配置信息

	git status  可以让我们时刻掌握仓库当前的状态
	git diff   查看修改的内容
	git log命令显示从最近到最远的提交日志
	git reset --hard HEAD^
	首先，Git必须知道当前版本是哪个版本，在Git中，用HEAD表示当前版本，也就是最新的提交
	1094adb..注意我的提交ID和你的肯定不一样），上一个版本就是HEAD^，上上一个版本就是
	HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100
	git reset --hard 版本号  可以跳转到指定版本号
	git reflog 记录每次操作命令的详细，有commitid值，可以让你直到回退到指定的任何版本
	的版本号

	穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。
	要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。）
	git tag -d v0.1 删除一个本地标签

问题解决：
Q: ! [rejected]        master -> master (non-fast-forward)
A:先使用 git pull --rebase origin master
然后再使用命令 push -u origin master

Q：输入git add readme.txt，得到错误：fatal: not a git repository (or any of the parent directories)。

A：Git命令必须在Git仓库目录内执行（git init除外），在仓库目录外执行是没有意义的。

Q：输入git add readme.txt，得到错误fatal: pathspec 'readme.txt' did not match any files。

A：添加某个文件时，该文件必须在当前目录下存在，用ls或者dir命令查看当前目录的文件，看看文件是否存在，或者是否写错了文件名。

Q:每次push就需要输入密码
A:	使用了https来连接远程origin ，更换ssh方式连接
1.git remote rm origin
2. git remote add origin git@github.com:xxxxx/xxxx.git
3.git push --set-upstream origin master
3. git push origin
